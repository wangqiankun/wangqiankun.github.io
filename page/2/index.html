git入门

发表于 2015-10-13   |  

###常用命令

  1. 从服务器down代码 ->git clone
  2. 查看提交纪录 ->git log
  3. 添加某个修改的文件 ->git add 修改的文件
  4. 把不想提交的文件放入.gitignore中 ->不知道
  5. add全部修改过的文件 ->git commit -a或git commit —all
  6. 更新代码 ->git pull -r（r表示rebase）
  7. 本地提交代码 ->git commit -m “这里是备注”
  8. 向服务器提交代码 ->git push（未确认）
  9. 把修改提交到gerrit中repo的master分支上（如果不是master分支，refs/for/分支名） ->git push origin HEAD:refs/for/master
  10. 本地纪录提交审核不通过的代码（git add 修改的文件） ->git commit —amend

使用友盟自定义错误

发表于 2015-10-13   |  

因为catch的Error或Exception友盟无法获取，那么就可以使用友盟提供的自定义错误来解决

  1. 在AndroidManifest.xml里面添加读取日志的权限android.permission.READ_LOGS
  2. 在MainActicity的OnCreate方法中调用
    MobclickAgent.setCatchUncaughtExceptions(true);
  3. 在catch的语句中使用用MobclickAgent.reportError(Context context,String error) 或者
    MobclickAgent.reportError(Context context, Throwable e)语句，使友盟上传日志

也可以自定义标签
MobclickAgent.onEvent(Contextcontext, String event_id, String label);

参考MobclickAgent——友盟用户分析工具

TextView中的滚动效果

发表于 2015-10-13   |  

只需要要在xml中添加以下字段
android:scrollbars=”vertical” //垂直 可以其他滚动方向

也可以扩展以下配置
android:scrollbarStyle=”X” //其中X为outsideOverlay或insideOverlay。
android:scrollbarFadeDuration=”X” //其中X表示滚动条从出现到消失的时间，单位是毫秒。

android:fadingEdge=”vertical” //设置拉滚动条时 ，边框渐变的方向。
